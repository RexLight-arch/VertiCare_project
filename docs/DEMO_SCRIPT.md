# VertiCare 答辩演示流程

这份流程用于现场演示。建议先让 ESP32、OneNet、Java Bridge、Qt 上位机全部运行稳定，再按顺序展示。

## 1. 启动检查

1. 打开 Arduino 串口监视器，波特率 115200。
2. 观察启动日志中 `[BOOT] VertiCare hardware summary`，确认引脚摘要、RFID 卡数量和采样周期已打印。
3. 等待 WiFi connected、OneNet MQTT connected。
4. 打开 Qt 上位机，确认右上角显示“设备在线 · 数据实时”。
5. 查看 Qt 总览页：温度、湿度、亮度、雨滴、维护状态、灌溉状态都有数据。

## 2. 自动灌溉演示

1. 保持控制页为自动模式。
2. 将开启阈值设置得高于当前湿度，方便触发灌溉。
3. 点击应用控制，观察 SG90 转到 90 度。
4. Qt 总览页显示“灌溉状态：运行中”。
5. 将关闭阈值调低或恢复正常阈值，演示系统自动停止。

讲解点：系统根据空气湿度和雨滴状态决定是否开启灌溉，避免无效浇水。

## 3. 雨滴抑制演示

1. 让雨滴检测板处于干燥状态，确认 Qt 显示“干燥”。
2. 用湿手或水滴轻触雨滴板。
3. Qt 显示“有雨”，事件页出现“检测到雨滴，自动灌溉将保持关闭”。
4. 自动模式下舵机应保持关闭或从开启变为关闭。

讲解点：雨滴检测不是单看瞬时值，而是相对干燥基线和连续确认，降低误触发。

## 4. RFID 授权维护演示

1. 长按 TTP223 约 3 秒进入录卡模式。
2. 刷 RFID 卡，Qt 人员页显示已录入人数增加，最近刷卡更新。
3. 再次刷已授权卡，Qt 显示“已授权”，授权剩余时间开始倒计时。
4. 敲击或晃动 SW-420，让维护状态显示“授权维护”。

讲解点：系统区分普通震动和维护行为，授权卡能证明当前维护人员身份。

## 5. 未授权维护/异常演示

1. 不刷授权卡或等待授权倒计时归零。
2. 敲击 SW-420 触发维护活动。
3. Qt 总览页维护状态显示“未授权维护”，事件页记录告警。

讲解点：震动传感器不是一触发就报警，而是连续活动窗口确认后才判断为维护活动。

## 6. 安全告警演示

1. 触发 SW-520D 倾斜，Qt 安全页显示“倾斜”，顶部状态变为安全告警。
2. 触发火焰传感器，Qt 安全页显示“火焰”，事件页记录高危事件。
3. 对 MQ-135 吹气，通风指数升高，空气状态可能从“清新”变为“建议通风”。

讲解点：安全告警、维护事件和环境数据统一进入 OneNet 物模型，并在上位机汇总显示。

## 7. 收尾展示

1. 打开事件页，展示系统概况、最近事件、维护人员和灌溉策略。
2. 切换人员页，展示 RFID 授权状态和已录入人数。
3. 切换控制页，展示 Qt 能通过 OneNet 远程下发控制属性。

推荐结论：本 demo 已完成传感器采集、云平台通信、上位机展示、远程控制、身份认证和安全事件闭环。
