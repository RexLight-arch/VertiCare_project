# VertiCare 发布说明

## v1.6-local-history

定位：Qt 本地数据中心版本。

### 主要新增

- Qt 上位机接入本地 SQLite，不需要外部数据库软件。
- 收到 OneNet 遥测后保存到 `data/verticare.db`。
- 程序启动时读取最后一条本地遥测；OneNet 断线时可先显示最后数据。
- 新增“趋势”页面，自绘最近 30 分钟温度、湿度、亮度和通风指数曲线。
- 新增 CSV 导出，可导出最近最多 5000 条遥测记录。
- 事件记录同步保存到 SQLite，并在启动时恢复最近历史事件。
- 新增参数设置页面，可在界面内保存 OneNet、Bridge 和刷新周期配置。
- Bridge 支持开机自动启动配置项 `autoStartBridge`。

## v1.5-polish

定位：课程设计稳定展示版。该版本不新增高功耗硬件，重点整理答辩展示、系统稳定性和工程交付材料。

### 主要能力

- ESP32-C6 采集温湿度、光照、雨滴、空气质量、振动、火焰、倾斜和 RFID 状态。
- SG90 模拟滴灌阀门，支持自动灌溉和手动灌溉。
- OneNet 物模型承载环境、安全、人员、事件和控制属性。
- Java Bridge 消费 OneNet 服务端订阅，并将数据推送给 Qt 上位机。
- Qt 上位机分为总览、安全、人员、控制、事件页面。
- RFID 与 TTP223 支持维护人员录卡、认证和本地交互。
- 事件中心支持雨滴、维护、安全、空气质量等事件展示。

### 本轮新增

- Qt 事件页新增系统概况、最近事件、维护人员、灌溉策略摘要。
- ESP32 启动日志新增 `[BOOT]` 硬件和版本摘要。
- 新增最终硬件接线表、答辩演示流程、常见问题排查、OneNet 架构说明和验收清单。
- `.gitignore` 增加密钥临时文件兜底规则，降低误上传风险。
- Qt 和 ESP32 统一显示版本号 `v1.5-polish`。

### 已知限制

- MQ-135 当前用于趋势展示和通风指数，不做专业 ppm 标定。
- SG90 用于模拟滴灌阀门，不直接驱动真实水泵。
- Qt 依赖 Java Bridge 接收 OneNet 服务端订阅，同一 Failover 订阅建议只运行一个上位机实例。
- Arduino 编译主要通过 Arduino IDE 验证；命令行环境可能没有 `arduino-cli`。

## v1.4-rfid-maintenance

定位：RFID 维护身份认证版。

### 主要新增

- 接入 RC522 RFID 和 TTP223。
- 支持录卡、授权识别、授权剩余时间和人员状态上报。
- 维护状态从单次震动升级为连续活动窗口判断。
- Qt 增加人员页，展示录卡模式、认证状态、已录入人数、当前人员和最近刷卡事件。

## v1.3-safety-events

定位：安全告警和事件中心版。

### 主要新增

- 接入 MQ-135、火焰传感器和 SW-520D 倾斜传感器。
- 增加空气质量状态、火焰告警、倾斜告警和综合安全告警。
- 增加事件冷却和 Qt 事件中心展示。

## v1.0-demo

定位：基础云端监测与灌溉控制 demo。

### 主要能力

- DHT11、光敏、雨滴、SW-420、SG90。
- ESP32 通过 OneNet MQTT 上报属性。
- Qt 通过 OneNet 获取数据并下发灌溉控制。
